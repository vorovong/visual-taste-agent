# Visual Taste Agent

비디자이너의 시각적 취향을 발견하고, 디자인 언어로 체계화하는 시스템.

## 왜 만들었나

좋은 디자인을 보면 "좋다"는 건 안다. 하지만 **왜 좋은지** 디자인 언어로 설명할 수 없다. AI 도구(V0, Bolt 등)로 UI를 생성해도 결과가 제네릭하다 -- "내 UI"가 아니다.

핵심 병목은 **생성이 아니라 평가**다. "별론데 왜인지 모름" 상태를 "정보 밀도가 높아서 별로"로 끌어올리는 것. 비평 AI보다 **취향 추출**이 먼저다.

레퍼런스를 모으고, 좋고 싫음을 판단하고, 그 판단들에서 패턴을 뽑아내면 -- 결국 나만의 디자인 시스템이 된다.

## 어떻게 쓰나

### 기본 흐름

1. **수집**: 텔레그램 봇에 URL을 보낸다. 5초면 끝난다. 봇이 자동으로 스크린샷을 찍고, 색상/폰트/레이아웃 같은 디자인 정보를 추출해서 DB에 저장한다. 이때 평가는 하지 않는다 ("미평가" 상태로 저장).
2. **평가**: 웹앱에 접속하면 갤러리 형태로 모아둔 레퍼런스를 볼 수 있다. 각 카드에서 좋아요/싫어요/삭제를 선택하고, 해시태그를 붙인다.
3. **분석**: 평가 데이터가 쌓이면 에이전트가 패턴을 추출한다. "이 사람은 여백 넓은 쇼핑몰을 좋아하는구나" 같은 인사이트를 뽑아낸다.

**핵심 원칙: 수집은 빠르게, 평가는 나중에.** 양이 먼저, 판단은 나중.

### 시스템이 성장하는 과정

데이터가 쌓이면 시스템이 단계적으로 똑똑해진다.

- **처음 (레퍼런스 0~4개)**: 빈 상태. "레퍼런스를 보여주세요."
- **5개 이상 평가**: 해시태그 제안이 시작된다. 자주 쓰는 태그를 먼저 보여줘서 입력 부담을 줄인다.
- **20개 이상 평가**: 패턴이 안정화된다. 갤러리에 그룹 탭(목적별/스타일별)이 자동으로 생긴다.
- **패턴 안정화 이후**: 디자인 토큰(색상, 간격, 폰트 등 수치화된 규칙)을 제안한다.
- **토큰 확정 이후**: 비평 모드 -- "이 UI를 내 취향으로 평가하면?" 같은 질문에 답할 수 있다.
- **최종 수렴**: 완성된 디자인 시스템이 생성된다. "이걸로 작업해줘"가 가능해진다.

### 사용 시나리오

**시나리오 1: 이동 중 레퍼런스 수집**

> 카페에서 노트북으로 작업하다 괜찮은 사이트를 발견. 텔레그램 봇에 URL을 보낸다. "저장됨. (47개, 미평가 4개)" 응답이 오면 끝. 나중에 웹앱에서 천천히 평가한다.

**시나리오 2: 주말 배치 평가**

> 웹앱에 접속하면 대시보드에 "미평가 12개"가 보인다. "지금 평가하기"를 누르면 미평가 레퍼런스만 모아서 보여준다. 좋아요/싫어요를 탭하고 해시태그를 선택한다. 해시태그 풀이 성숙하면 탭만으로 완료 -- 타이핑이 필요 없다.

## 구조

```
[텔레그램봇]                    [Next.js 웹앱]
 수집: URL -> 캡처               열람 + 평가: 갤러리, 상세
 + 메타추출 -> DB                Google OAuth 인증
       |                              |
       v                              v
  +-----------+                 +-----------+
  | SQLite DB | <============> | SQLite DB |
  | (단일 파일) |    같은 파일     | (단일 파일) |
  +-----------+                 +-----------+
       ^
       |
  [Claude Code 에이전트]
   패턴 추출, 디자인 시스템 생성
```

- **텔레그램봇**: 수집 전담. URL을 받으면 Puppeteer로 스크린샷을 찍고 디자인 메타데이터(색상, 폰트, 레이아웃)를 추출해서 DB에 저장한다. 별도 프로세스로 상시 실행.
- **웹앱**: 열람 + 평가 전담. 갤러리에서 레퍼런스를 보고, 상세 페이지에서 멀티뷰포트(데스크톱/태블릿/모바일) 프리뷰를 확인하고, 좋아요/싫어요 + 해시태그로 평가한다.
- **에이전트**: 분석 전담. Claude Code Agent로 실행. DB에서 평가 데이터를 읽어 패턴을 추출하고, 최종적으로 디자인 시스템을 생성한다.

세 컴포넌트 모두 하나의 SQLite DB 파일(`data/vta.db`)을 공유한다.

## 기술 스택

| 기술 | 역할 | 설명 |
|---|---|---|
| Next.js 15 | 웹앱 프레임워크 | React 기반 풀스택 웹 프레임워크. App Router 사용 |
| Tailwind CSS v4 | 스타일링 | HTML 안에서 바로 스타일을 지정하는 유틸리티 CSS 도구 |
| TypeScript | 타입 시스템 | JavaScript에 타입 검사를 추가해서 버그를 줄이는 언어 |
| NextAuth v5 | 인증 | Google 계정으로 로그인하는 기능을 처리. 허용된 이메일만 접근 가능 |
| SQLite | 데이터베이스 | 파일 하나로 동작하는 가벼운 데이터베이스. 별도 서버 불필요 |
| Drizzle ORM | DB 접근 도구 | TypeScript 코드로 DB를 조회/수정할 수 있게 해주는 라이브러리 |
| Grammy | 텔레그램봇 | 텔레그램 봇 API를 쉽게 사용할 수 있게 해주는 라이브러리 |
| Puppeteer | 캡처 + 메타추출 | 브라우저를 코드로 조종해서 스크린샷을 찍고 CSS 정보를 추출 |
| Playwright | E2E 테스트 | 실제 브라우저에서 사용자 시나리오를 자동으로 테스트하는 도구 |
| Fly.io | 배포 | Docker 컨테이너 기반 클라우드 호스팅. Tokyo 리전 |

## 시작하기

### 사전 준비

- Node.js (v18 이상)
- 텔레그램 봇 토큰 (@BotFather에서 발급)
- Google OAuth 클라이언트 ID/Secret (Google Cloud Console에서 발급)

### 실행 단계

```bash
# 1. 환경변수 설정
cp .env.example .env
# .env 파일을 열어서 아래 값을 채운다:
#   TELEGRAM_BOT_TOKEN, ALLOWED_CHAT_ID,
#   GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET,
#   ALLOWED_EMAIL, NEXTAUTH_SECRET

# 2. 의존성 설치
npm install

# 3. DB 초기화 (SQLite 파일 생성 + 테이블 생성)
npm run migrate

# 4. 웹앱 실행 (localhost:3000)
npm run dev

# 5. 텔레그램봇 실행 (별도 터미널에서)
npm run bot
```
