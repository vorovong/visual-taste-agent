# Visual Taste Agent — 마일스톤 + 실행 순서 (planning.md)

---

## 마일스톤 개요

```
M1: 프로젝트 초기화 + 데이터 구조
M2: 텔레그램봇 (수집 채널)
M3: Puppeteer 캡쳐
M4: 에이전트 정의 + 레퍼런스 분석
M5: 자동 레벨업 + 패턴 추출
M6: 토큰화 + 비평 모드
M7: 통합 테스트 + 안정화
```

---

## M1: 프로젝트 초기화 + 데이터 구조

**목표**: 프로젝트 뼈대, git, 데이터 폴더, 빈 profile.md

- [ ] git init + .gitignore 설정
- [ ] package.json 초기화 (Node.js)
- [ ] .env.example 작성 (TELEGRAM_BOT_TOKEN, CLAUDE_API_KEY 등)
- [ ] design-system/ 폴더 구조 생성
- [ ] profile.md 빈 상태 초기화 (확실한 것 / 추정 / 거부 이력 / 맥락별 섹션)
- [ ] 레퍼런스 템플릿 (references/TEMPLATE.md)

---

## M2: 텔레그램봇 (수집 채널)

**목표**: URL/이미지/파일 수신 → 맥락 태깅 → 판정 → 로컬 저장

- [ ] 텔레그램 봇 토큰 발급 방법 안내 (사용자 액션 필요)
- [ ] 봇 엔트리포인트 (bot/index.js)
- [ ] 메시지 수신 핸들러 (URL 감지, 이미지 다운로드, 파일 다운로드)
- [ ] 인라인 키보드: 맥락 선택 (매체/용도/청중)
- [ ] 인라인 키보드: 판정 (👍/👎)
- [ ] 레퍼런스 파일 생성 (YAML 프론트매터 + 기본 분석)
- [ ] 봇 실행 스크립트 (npm run bot)

**참조**: `~/projects/telegram-bot-api/` 기존 봇 구조

---

## M3: Puppeteer 캡쳐

**목표**: URL → 뷰포트별 스크린샷 자동 생성

- [ ] puppeteer 설치 + 설정
- [ ] scripts/capture.js (URL → 3종 뷰포트 캡쳐)
- [ ] 캡쳐 결과를 references/{id}/screenshots/에 저장
- [ ] 텔레그램봇과 연동 (URL 수신 시 자동 캡쳐 트리거)
- [ ] 캡쳐 실패 시 폴백 (스크린샷 직접 전송 유도)

---

## M4: 에이전트 정의 + 레퍼런스 분석

**목표**: `claude --agent design-system`으로 심층 분석 가능

- [ ] .claude/agents/design-system.md 작성 (에이전트 정의)
- [ ] 에이전트가 design-system/ 데이터에 접근하는 구조
- [ ] 레퍼런스 파일 읽기 → 시각 요소 분석 (색상, 여백, 서체, 레이아웃, 밀도)
- [ ] 분석 결과를 레퍼런스 파일에 기록
- [ ] profile.md 자동 업데이트 ("추정" 영역)

---

## M5: 자동 레벨업 + 패턴 추출

**목표**: 레퍼런스 수에 따라 자동 성장, 패턴 추출 + 반박 루프

- [ ] 레벨 판단 로직 (레퍼런스 수, 패턴 안정도 기반)
- [ ] Lv.1: 5개+ 레퍼런스 → 패턴 추출 → patterns/ 생성
- [ ] 패턴 제시 → 사용자 반박 → profile.md "거부 이력" 기록
- [ ] Lv.2: 패턴 안정화 → 토큰 제안
- [ ] 텔레그램봇에 현재 레벨 표시

---

## M6: 토큰화 + 비평 모드

**목표**: 취향 → 디자인 토큰 → 시스템 → 비평

- [ ] Lv.2: 토큰 제안 (core.md 초안)
- [ ] Lv.3: 토큰 확정 → contexts/ 맥락별 변주
- [ ] Lv.3: system/ 문서 생성 (컴포넌트 가이드)
- [ ] Lv.4: 비평 모드 — 새 디자인 입력 → 취향 일관성 판단
- [ ] Lv.4: 청중 적합성 판단 (디렉터 × 프로듀서)

---

## M7: 통합 테스트 + 안정화

**목표**: 전체 흐름 end-to-end 동작 확인

- [ ] 텔레그램 → 캡쳐 → 저장 → 분석 → 패턴 추출 전체 흐름
- [ ] 에이전트 세션에서 프로필 리뷰 → 반박 → 수정 흐름
- [ ] 엣지 케이스 처리 확인
- [ ] 봇 안정성 (장시간 실행, 에러 핸들링)

---

## 실행 순서

```
M1 → M2 → M3 → M4 → M5 → M6 → M7
          ↑                   ↑
     사용자 액션 필요      품질 단계
     (봇 토큰 발급)
```

**M1~M3**: MVP 핵심 (수집 채널 완성)
**M4~M5**: 에이전트 지능 (분석 + 패턴)
**M6~M7**: 완성 (토큰화 + 비평)

---

## 기술 스택 (결정됨)

| 영역 | 기술 | 이유 |
|---|---|---|
| 런타임 | Node.js | Puppeteer 네이티브, 텔레그램 라이브러리 풍부 |
| 텔레그램 | node-telegram-bot-api 또는 grammy | [구현 시 결정] |
| 캡쳐 | Puppeteer | 뷰포트별 스크린샷, headless 브라우저 |
| 에이전트 | Claude Code custom agent | .claude/agents/ 네이티브 |
| 데이터 | 로컬 마크다운 + Git | 지식 저장소와 동일 패턴, 이식성 |
